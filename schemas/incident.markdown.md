# ModCom Incident — Markdown schema (v1.0)

Canonical import format. Human-readable. What AIs should produce for **Import from AI**.

## Required document shape

```markdown
## ModCom Incident

**Title:** <string, max 255 chars>
**Severity:** <critical|high|medium|low>
**Classification:** <general|safety|environmental|security|operational|quality|hr|financial>
**Date Detected:** <YYYY-MM-DD>
**Source:** <free text>
**Location:** <optional, max 255>
**Status:** AI-generated draft — awaiting review

### Description
...

### Affected Systems
- ...

### Immediate Actions Taken
- ...

### Control Gaps Identified
- ...

### Recommended Next Steps
- [ ] ...

### Provenance
**Generated by:** AI  
**Model / tool:** <free text>  
**Generated at:** <ISO-8601 optional>  
**Human review:** required before live status  
```

## Enums (must match product)

| Field | Values |
|-------|--------|
| severity | `critical`, `high`, `medium`, `low` |
| classification | `general`, `safety`, `environmental`, `security`, `operational`, `quality`, `hr`, `financial` |

Import always forces product status **`ai_draft`** regardless of other wording.

## Field map → Modular Compliance product

| Markdown | Product |
|----------|---------|
| Title | `incidents.title` (required) |
| Severity | `incidents.severity` |
| Classification | `incidents.classification` (default `general`) |
| Date Detected | Note in description if ≠ create time; `reported_at` policy is product-side |
| Source | `import_source` + description header |
| Location | `incidents.location` |
| Body sections | Composed into `incidents.description` |
| Provenance | `ai_generated`, `ai_model`, `ai_raw_markdown`, later `reviewed_by` / `reviewed_at` |
| Status line | Forced to `ai_draft` on insert |

## Parser rules (product)

1. Heading `## ModCom Incident` required (case-insensitive).
2. Unknown bold keys ignored.
3. Missing severity → default `medium` + warning.
4. Invalid enum → error; block save until fixed.
5. Empty title → block save.
6. Preserve original paste as raw audit payload.
